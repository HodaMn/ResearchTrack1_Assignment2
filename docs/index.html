node_a module

Module author: Hoda Mostafanezhad 5806506@studenti.unige.it

A node that implements an action client, allowing the user to set a target (x, y) or to cancel it. The node also publishes the robot position and velocity as a custom message (x,y, vel_x, vel_z), by relying on the values published on the topic /odom;

scripts.node_a.check_for_stop(client)[source]

Function for cancelling the movement or exit the program

scripts.node_a.clbk_odom(msg)[source]

Callback function for ‘/odom’ topic. Updates the position and velocity information.

Args: msg(Info): the robot’s position and velocity

scripts.node_a.get_info_goal(req)[source]

Service callback function for ‘goal_info’ service. Returns the target reached and target canceled counts.

scripts.node_a.main()[source]

Main function of node_a. - Initializes ROS node and necessary elements. - Sends goal requests to the server. - Handles user input for canceling the target. - Prints the target reached and target canceled counts.

node_c module

Module author: Hoda Mostafanezhad 5806506@studenti.unige.it

Anotherservice node thatsubscribes to the robot’s position and velocity (using the custom message) and implements a server to retrieve the distance of the robot from the target and the robot’s average speed.

scripts.node_c.clbk_info(msg)[source]

Callback function for the ‘/bot_info’ topic subscriber. Calculates the average velocity and distance from the current position to the target position.

scripts.node_c.clbk_tgt(msg)[source]

Callback function for the ‘/tgt’ topic subscriber. Updates the target position.
